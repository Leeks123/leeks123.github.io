### 거짓 값이 있는 조건문을 축약하라

```javascript
1 == '1' // true. 동등한지

1 === '1' // false. 일치하는지
1 === 1  // true
```

###### 거짓 값들

false, null, 0, NaN, '', ""

빈 배열과 빈 객체는 항상 참 값. 따라서 비어있는지 확인하려면 length함수를 활용한 숫자확인이 방법

### 삼항 연산자로 빠르게 데이터를 확인하라

삼항연산자를 활용하면 단순한 코드와 더불어 변수의 재할당을 줄일 수 있음

If, else문을 활용할 경우 블록 유효 범위 변수를 고려해야 하며 변수를 재할당하는 경우가 있음

삼항연산자를 여러 개 연결하는 것은 피해야 함.
가시성도 떨어지면서 단순함이라는 가치도 사라짐. 코드가 지나치게 모호해진다면 if문을 사용하는 것이 필요

### 단락 평가를 이용해 효율성을 극대화하라

단락 평가의 목적은 가장 타당한 정보를 먼저 위치시켜서 정보 확인을 건너뛰는 것

```javascript
const userConfig = {
  image: []
}

function getImage(userConfig) {
  let img = 'default.png';
  if (userConfig.images) {
    if (userConfig.images.length) {
      return userConfig.images[0];
    }
  }
  return img;
}
```

단락평가로 깔끔하게

```javascript
function getImage(userConfig){
  if(userConfig.images && userConfig.images.length > 0){
    return userConfig.images[0];
  }
  return 'default.png';
}
```

TypeError는 피할 수 있지만 신뢰할만한 데이터이거나 데이터 정규화의 과정이 필요하다.

단락평가와 삼항연산자로 조합하면

```javascript
function getImage(userConfig){
  const images = userConfig.images;
  return images && images.length ? image[0]:'default.png';\
}
```
