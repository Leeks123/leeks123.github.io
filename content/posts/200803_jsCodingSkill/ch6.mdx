### 매개변수 기본값을 생성하라

```javascript
function convertWeight(weight, ounces, roundTo) {
  const oz = ounces / 16 || 0;
  const total = weight + oz;
  const conversion = total / 2.2;
  const round = roundTo === undefined ? 2 : roundTo;
  return roundToDecimalPlace(conversion, round);
}
```

단란평가로 매개변수의 디폴트 값을 지정한 경우

```javascript
function convertWeight(weight, ounces = 0, roundTo = 2) {
  const total = weight + (ounces / 16);
  const conversion = total / 2.2;
  return roundToDecimalPlace(conversion, roundTo);
}
```

- 함수 사용자가 매개변수를 누락하더라도 어떤 값이든 반환될 것이라고 확신할 수 있음
- 특정한 자료형이 필요하다는 단서를 다른 개발자에게 알려줄 수도 있음
- 매개변수는 항상 순서를 지켜야 함

### 해체 할당으로 객체 속성에 접근하라

해체할당이라는 과정을 통해 객체에 있는 정보를 곧바로 변수에 할당할 수 있음

동작원리는 먼저 객체에 있는 키와 같은 이름의 변수를 생성하고, 객체에 있는 키에 연결된 값을 생성한 변수의 값을 할당

```javascript
const landscape = {
  photographer: 'Nathan',
  equipment: 'Canon',
  format: 'digital',
};

const {photographer} = landscape;
photographer;
// Nathan

const {
  photographer,
  ...additional
} = landscape;

additional;
// { equipment:'Canon', format: 'digital'}
```

### 나머지 매개변수로 여러 개의 인수를 변수로 전달하라

###### every

every( ) 메서드는 배열의 모든 항목을 대상으로 콜백 함수를 실행해 모든 항복에서 참 값이 반환되면 결과적으로 true를 반환

```javascript
function validateCharacterCount(max, items) {
  return items.every(item => item.length < max);
}

validateCharacterCount(10,['Hobbs','Eagles']); 
// true
validateCharacterCount(10,'wvoquine');
// TypeError : items.every is not a function

const tags = ['Hobbs','Eagles'];
validateCharacterCount(10,...tags); 
// true
```

```javascript
function validateCharacterCount(max, ...items) {
  return items.every(item => item.length < max);
}
```

간결하고 예측 가능성도 높아짐. 한눈에 두 개 이상의 인수를 받을 수 있다는 것을 알 수 있음

###### 나머지 인수를 사용하는 이유

1. 인수를 배열로 다루는 것을 다른 개발자들에게 알려야 하는 경우
2. 나머지 매개변수는 코드 디버깅에 좋은 방법이 될 수 있음
3. 나머지 인수는 함수 간에 속성을 전달하면서 해당 속성을 조작할 필요가 없을 때 사용하면 좋음
